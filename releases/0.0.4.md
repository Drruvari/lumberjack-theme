# 0.0.4 - 2026-03-04

## Summary

Release automation and theme-quality update: added dedicated default variation, ensured all variants are unique, resolved TypeScript diagnostics, and automated publish flow.

## Highlights

- Added true `default` theme variation and mapped alias file to default.
- Every VS Code variant now generates unique output.
- Auto-publish workflow now handles version bump, package, release assets, and marketplace publish.

## Added

- `default` variation (`src/variations/default.ts`).
- `src/types/node-shims.d.ts` for local type diagnostics when Node typings are unavailable.
- GitHub workflow enhancements for VSIX artifacts and release uploads.

## Changed

- `VariationId` extended with `default`.
- Build order updated to include `default` first.
- VS Code generator alias (`lumberjack-color-theme.json`) now comes from `default`.
- Default theme styling aligned with Bearded/WebDevCody-inspired palette.

## Fixed

- TS5097 import-extension diagnostics across project config.
- Node module type-resolution diagnostics in local environment.
- Duplicate dark/default visual output issue.

## Breaking Changes

- None

## Verification

- `npm run typecheck`
- `npm run build`
- `npm run validate:engine`
- Verified unique checksums for all VS Code variants.
