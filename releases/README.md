# Releases

This directory contains clean, detailed release notes.

## Format

- One version per file: `<version>.md`
- Example: `0.0.4.md`
- Keep entries factual and testable.
- CI auto-generates new files via `npm run release:auto-note -- <version>`.

## Required Sections

1. `Summary`
2. `Highlights`
3. `Added`
4. `Changed`
5. `Fixed`
6. `Breaking Changes` (if any)
7. `Verification`
8. `Assets`
